设计说明

流程修订（相对 SPEC v0.2 的 Spec Review 结论）

① Step 1 收敛为官网工具的三个原生字段（Country of passport / Education provider / Student type），移除姓名字段——文档标题相应不再含姓名，隐私面进一步收紧（App 全程不采集个人信息，SPEC §7 标题格式与 §9 隐私条款需同步更新）；② 原「生成进度视图」降为轻量的 Step 2：用户只看到「搜索官网清单 → 翻译成中文」两个阶段与当前 provider / fallback 提示；③ 判定、抓取、解析、分类、备注注入等细粒度过程改为写入 设置 → 日志，供排障与中介顾问审计，不再打断主流程。免责声明、抓取时间、清单类型三要素不受影响，仍在一切结果展示中保留。

配色

严格限定在天蓝、浅灰、白三色系：桌面底 #E9EDF2、窗口 #F5F7FA、卡片纯白 #FFFFFF，天蓝 #2E9BDF（深色模式提亮为 #4FB0F0）只用于「可交互 / 当前进行 / 官方术语」三种语义，保持工具型应用的克制。此外仅保留两个功能色：琥珀色用于降级与配额告警，红色仅用于 R3 公证翻译警告与错误——签证材料是严肃输出，警示色出现得越少，出现时越有分量。深浅两套色板通过 CSS 变量 + prefers-color-scheme 切换，无 JS 参与。

字体层级

系统字体栈 -apple-system / PingFang SC，正文 13–14.5px、辅助信息 11–12.5px，标题不超过 21px；数字（耗时、CRICOS 码）用 tabular-nums 对齐。层级主要靠字重（400/600/700）与灰度（三级文字色）表达，而非字号跳跃，维持 macOS 原生密度。

双语对照方案的取舍

核心决策是「中文为主、英文按需」的两层结构，而非左右分栏：分栏会把每条材料压成两列窄栏，中文可读性受损，且英文原文长短不一会造成大量空白。最终方案为——中文译文作为条目主行；官方术语在主行内以蓝色括注保留英文（CoE、OSHC、GS、Form 956），因为这些词用户在官网和申请系统里终究要面对英文；完整英文原文折叠在每条目下方的「英文原文」展开层（左侧蓝边引用样式，附官网原文链接），支持逐条展开与标题栏「展开全部英文」全局开关，兼顾中介顾问核对原文与普通申请人快速通读两种场景。备注体系与翻译层视觉分离：普通备注（R1/R2）为灰底小签，R3 公证翻译警告为红色系高对比条，确定性规则产出的固定文案不与 AI 译文混排。

红线自查（AGENTS.md §5）

所有结果类界面（03 结果视图、05 状态 D 未翻译降级视图）均完整保留三要素：抓取时间、清单类型徽章（Regular / Streamlined / Undetermined）、底部固定免责声明；元信息同时记录实际使用的 AI provider 与模型。
